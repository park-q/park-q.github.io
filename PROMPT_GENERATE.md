# PROMPT_GENERATE — 글 생성 규칙 (재사용 템플릿)

> 사용법: 이 파일을 클로드 코드에게 그대로 보여주면서
> **"priority 1~N번 규칙대로 `{keyword}` 글 만들어줘"** 라고 지시하면 된다.
> 변수 자리(`{keyword}`, `{seed}`, `{intent}`, `{suggested_title}`, `{notes}`)는
> 지시할 때 실제 값으로 채워서 전달한다.

---

## 입력 변수

| 변수 | 의미 | 예시 |
| --- | --- | --- |
| `{keyword}` | 핵심 검색 키워드(글의 주제) | `월배당 ETF 세금` |
| `{seed}` | 출발점이 된 씨앗 키워드/주제군 | `월배당 ETF` |
| `{intent}` | 검색 의도(아래 priority 7 참고) | `정보형 - 세금 구조 이해` |
| `{suggested_title}` | 제안 제목(없으면 직접 작성) | `월배당 ETF 세금 계산, 분배금 과세 한눈에` |
| `{notes}` | 이 글에만 적용할 특이사항·강조점·피해야 할 내용 | `2026 한시 분리과세 제도 반드시 언급` |

---

## Priority 1 — 파일 위치·이름(슬러그) 규칙

- 저장 위치: `content/posts/{slug}.md`
- `{slug}`는 **영문 소문자 + 하이픈(kebab-case)**, 키워드를 영문화한 형태.
  - 예: `월배당 ETF 세금` → `monthly-dividend-etf-tax`
  - 예: `배당락 손해 안 보는 법` → `dividend-ex-date-no-loss`
- 슬러그는 짧고 의미가 드러나게. 날짜·숫자 접미사 금지.

## Priority 2 — 같은 슬러그 건너뛰기

- 글을 만들기 전에 `content/posts/{slug}.md`가 **이미 있는지 먼저 확인**한다.
- 이미 존재하면 **새로 만들지 말고 건너뛴다**(덮어쓰기 금지). 건너뛴 사실을 보고한다.
- 여러 글을 한 번에 생성할 때도 슬러그 단위로 중복을 검사한다.

## Priority 3 — Front matter 형식 (TOML, `+++`)

```toml
+++
date = '{YYYY-MM-DD}T{HH:MM:SS}+09:00'
draft = true
title = '{suggested_title}'
categories = ['{대분류 1개}']
tags = ['{태그1}', '{태그2}', '{태그3}', '{태그4}']
description = '{검색결과에 노출될 1~2문장 요약. 키워드 포함, 초보 눈높이}'
+++
```

- `draft = true`가 **기본값**이다(발행은 사람이 검토 후 별도로 `false`로 전환).
- `date`는 **생성 시점 이전**의 시각으로 둔다. 미래 시각이면 Hugo가 빌드에서 제외해 글이 안 보인다(`buildFuture = false`). 여러 글을 연속 생성하면 10분 간격 등으로 순서만 유지.
- `categories`는 1개, `tags`는 3~5개.

## Priority 4 — 본문 첫 줄에 `# 제목` 넣지 않기

- 제목은 front matter `title`이 담당한다. 본문 맨 위에 `# ...` 마크다운 H1을 **절대 넣지 않는다**(중복 제목 방지).
- 본문 소제목은 `##`(H2)부터 시작한다.

## Priority 5 — 두괄식 서론

- 본문 **첫 줄은 결론을 굵게**(`**...**`) 한 문장으로 제시한다(두괄식).
- 이어서 1~2문단으로 배경·범위를 풀어 쓴다.
- 관련 글이 있으면 서론 안에서 내부 링크로 연결한다: `[앵커 텍스트](/posts/{관련-slug}/)`.

## Priority 6 — 본문 구조(소제목·표·FAQ)

- **소제목(`##`) 4~6개**로 구성한다.
- **표(table) 최소 1개** 포함(비교·분류·요약에 활용).
- 글 끝부분에 **자주 묻는 질문(FAQ) 3~5개**:
  - 형식: `**Q. 질문?**` 다음 줄에 답변(2~3문장).
- 필요하면 단계 설명은 번호 목록(1. 2. 3.)으로.

## Priority 7 — 검색 의도별 중심 잡기

- `{intent}`에 맞춰 글의 무게중심을 둔다. 한 글이 모든 걸 다루려 하지 말고 **하나의 의도에 집중**한다.
  - **정보형**(개념·구조 이해): 원리·정의·구조 설명 중심. 표/흐름도로 구조화.
  - **방법형**(how-to): 순서·체크포인트 중심. 번호 목록 활용.
  - **비교형**: 비교 표를 핵심에 배치, 선택 기준 제시.
  - **시점·일정형**: 날짜·순서·타이밍 중심, 흐름도 적합.
- 같은 주제군(`{seed}`)에서 의도가 다른 글은 **서로 내부 링크**로 연결하되 내용 중복은 피한다.

## Priority 8 — 수치 단정 금지 · "확인 필요" 표시

- 세율·기준금액·일정 등 **변할 수 있는 수치는 단정하지 않는다.**
  - "흔히 ~로 알려져 있음", "일반적으로 ~", "그해 기준을 확인하세요" 같은 완충 표현 사용.
- 제도·요건처럼 케이스별로 갈리는 내용은 **"확인이 필요합니다"**를 명시한다.
- 중요한 주의는 인용블록 + ⚠️로 강조:
  > ⚠️ 세율·과세 기준은 바뀔 수 있습니다. 공식 자료(국세청·증권사 공지)로 확인하세요.

## Priority 9 — 투자 권유 금지 문구(맨 끝 고정)

- 본문 마지막에 구분선과 함께 **반드시** 아래 문구를 넣는다:

```markdown
---

> 본 글은 투자 권유가 아닌 정보 제공을 목적으로 합니다.
```

## Priority 10 — 톤·눈높이

- 독자는 **초보**다. 쉬운 한국어로, 단계적으로, 전문용어는 풀어서 설명한다.
- 단정·과장·수익 보장성 표현 금지. 차분하고 중립적인 어조.

---

## 산출물 체크리스트(생성 후 자가 점검)

- [ ] `content/posts/{slug}.md` 경로·kebab-case 슬러그
- [ ] 기존 동일 슬러그 없음(있으면 건너뜀)
- [ ] Front matter: `draft = true`, 과거 시각, category 1·tags 3~5, description
- [ ] 본문 첫 줄 `#` 제목 없음
- [ ] 두괄식 서론(첫 줄 굵게) + 관련 글 내부 링크
- [ ] `##` 소제목 4~6개
- [ ] 표 최소 1개
- [ ] FAQ 3~5개
- [ ] 검색 의도(`{intent}`)에 집중
- [ ] 수치 단정 없음 + "확인 필요" 표시
- [ ] 맨 끝 투자 권유 금지 문구
- [ ] (시각요소 보강은 `PROMPT_ENHANCE.md` 참고)
